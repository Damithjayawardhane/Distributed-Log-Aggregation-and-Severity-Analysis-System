Distributed log aggregation & Severity analysis

SE5090 · Distributed Computing

Overview

Cluster pipeline

  • Topology: one coordinator, multiple mapper workers, multiple validators (quorum), one aggregator.
  • Severity levels: INFO, DEBUG, WARNING, ERROR, CRITICAL.
  • Mapper: parses log lines; per-severity totals and distinct message lines.
  • Validator: independent recount on the raw chunk; vote uses floor(N/2)+1 ACCEPT of N validators.
  • Aggregator: merges accepted mapper totals; skips duplicate chunkId submissions.
  • Coordinator: splits the input file into line-range shards for mappers.
Run

POST /masterCoordinator/start

Input file: logs.txt next to the project (override with LogProcessing:LogFilePath in appsettings.json).
